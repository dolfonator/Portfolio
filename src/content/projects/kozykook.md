## Problem

Local restaurants often send customers across social feeds, food delivery apps, and chat threads before they can answer the basic questions: what is on the menu, where is the shop, and how do I order or book?

Kozykook needed a site concept that could gather those decisions into one fast page while still feeling like the restaurant: warm, bright, casual, and useful on a phone.

## Approach

The build uses a no-framework HTML/CSS/JS stack so the page can load quickly and stay easy to deploy. The structure follows the way a hungry customer scans: identity, highlights, menu, gallery, reviews, visit details, and booking.

The design system is token-driven. Brand colors, typography, spacing, section bands, panels, and button styles are centralized in CSS so the same pattern can become a repeatable package for future food clients without turning every site into a clone.

## Result

The result is a polished Package B-style business site: full menu on-site, review proof, map/location context, foodpanda routing, Messenger-aware decisions, and structured data for search engines.

Because this is a founding-client demo, the live page is intentionally marked as a concept and hidden from search indexing. The portfolio case study can still show the craft honestly: speed, clarity, rebrandable design tokens, and practical customer routing.

## Technical appendix

- Static HTML/CSS/JS keeps hosting simple across Vercel or Netlify.
- Restaurant JSON-LD gives search engines structured business context.
- CSS tokens separate the brand layer from the reusable layout system.
- The nav, reveal effects, and form/demo behavior are vanilla JavaScript.
- The demo uses a `noindex` tag because it is presentation work, not the official client site.
